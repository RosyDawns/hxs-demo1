## 一、项目概述

唤醒兽平台是基于 Vue 3 + Vite + TailwindCSS 的移动端生活服务应用，连接生活技能教练（"唤醒狮"）和用户。

### 技术栈
- **Vue 3** (v3.5.17) - Composition API
- **Vue Router** (v4.5.1) - 路由管理
- **Vite** (v7.0.4) - 构建工具
- **TailwindCSS** (v4.1.13) - 样式框架
- **高德地图 API** - 地图服务

## 三、底部导航结构

底部导航栏（FooterNav.vue）包含 5 个主要入口：

```
┌─────────────────────────────────────────────────────────┐
│  首页  │  大狮兄  │  能量  │  消息  │  我的  │
│   /    │/ai-assistant│/energy-tree│/messages│  /me  │
└─────────────────────────────────────────────────────────┘
```

**特殊交互：** 首页按钮支持双向切换
- 在首页(`/`)点击 → 跳转到唤醒狮列表(`/teacher-list`)
- 在唤醒狮列表点击 → 跳转回首页(`/`)

## 六、路由守卫

项目使用全局前置守卫实现：

1. **滚动位置保存/恢复** - 离开页面时保存滚动位置，返回时恢复
2. **登录态检查** - 需要认证的页面会检查 `localStorage.token`（当前已注释）

```javascript
// meta.requiresAuth = true 的路由需要登录
router.beforeEach((to, from, next) => {
  // 保存滚动位置
  // 登录守卫检查（当前已注释）
  next();
});
```

# 唤醒兽平台(HXS)项目路由结构与页面跳转文档

## 1. 项目概述

唤醒兽平台是一个基于 Vue 3 的移动端生活服务类应用，旨在为用户提供生活技能学习、教练推荐、AI助手等服务。平台连接生活技能教练("唤醒狮")和用户，打造一个技能分享和学习的生态平台。

## 2. 路由架构总览

项目采用 Vue Router 4 进行路由管理，路由配置按照功能模块进行拆分，主要包括以下模块：

- core.js - 核心页面路由
- camp.js - 唤醒营相关路由
- account.js - 账户与个人中心路由
- commerce.js - 商业与电商相关路由
- social.js - 社交与内容互动路由
- org.js - 组织与唤醒师相关路由
- help.js - 帮助与信息路由

## 4. 页面跳转关系分析

### 4.1 首页(IndexPage)跳转关系

首页作为平台的核心入口，提供了多个功能模块的跳转入口：

1. 顶部搜索框 → `/search` (搜索页面)
2. "运动健康数字地图"按钮 → `/map-demo` (地图演示页面)
3. 城市选择 → 弹出城市选择器
4. 核心功能区：
   - 生活主理人 → `/teacher-list?tab=主理人&top=discover`
   - 技能唤醒师 → `/teacher-list?tab=生活技能&top=discover`
   - 同城好搭子 → `/teacher-list?tab=生活搭子&top=discover`
5. 快捷入口：
   - 城市大师 → `/ranking` (排行榜页面)
   - 免费体验 → `/camp-register` (唤醒营注册页面)
   - 体重管理唤醒营 → `/weight-management-camp` (体重管理唤醒营页面)
   - 组团去玩 → `/group-play` (组团去玩页面)
6. 教练列表卡片 → `/ouyang` (欧阳老师详情页面)
7. 动态列表 → `/dynamic-detail/:id` (动态详情页面)

### 4.2 个人中心(MePage)跳转关系

个人中心是用户个人信息和功能的集中展示区域：

1. 添加好友按钮 → `/add-friends` (添加好友页面)
2. 设置按钮 → `/settings` (设置页面)
3. 功能卡片区：
   - 订单 → `/my-orders` (我的订单页面)
   - 收藏 → `/favorites` (我的收藏页面)
   - 待评价 → `/pending-review` (待评价页面)
   - 真实评价 → `/my-reviews` (真实评价页面)
4. 全部功能区：
   - 我的钱包 → `/wallet` (我的钱包页面)
   - 注册唤醒师 → `/register-waker` (注册唤醒师页面)
   - 醒师掌柜 → `/business-admin` (商家后台页面)
   - 唤醒联盟 → `/waker-alliance` (唤醒联盟页面)
   - 醒狮团队 → `/team` (醒狮团队页面)
5. 标签切换内容：
   - 动态内容 → `/dynamic-detail/:id` (动态详情页面)
   - 关注内容 → 点击教练跳转至 `/ouyang` (欧阳老师详情页面)
   - 橱窗内容 → `/product-detail/:id` (商品详情页面)
   - 推荐橱窗内容 → 点击教练跳转至 `/ouyang` (欧阳老师详情页面)
6. 发布按钮 → `/publish-activity` (发布动态页面)

### 4.3 教练列表(TeacherListPage)跳转关系

教练列表页面提供多种维度的教练筛选和展示：

1. 返回按钮 → 返回上一页
2. 顶部标签切换：
   - 发现 → 展示发现相关的教练列表
   - 城市 → 展示城市相关的教练列表
   - 关注 → 展示关注相关的教练列表
3. 教练列表项 → `/ouyang` (欧阳老师详情页面)
4. 动态列表项 → `/dynamic-detail/:id` (动态详情页面)

项目路由配置中包含了丰富的元信息，用于页面管理和用户体验优化：

- `title`: 页面标题
- `icon`: 页面图标
- `showInNav`: 是否在导航中显示
- `requiresAuth`: 是否需要登录验证

项目实现了智能的滚动位置管理：

- 保存和恢复页面滚动位置
- 处理嵌套跳转时的滚动位置
- 提供良好的用户体验

虽然目前登录守卫被注释掉，但项目保留了登录状态控制的实现：

- `isAuthenticated()` 函数用于判断用户登录状态
- 可通过路由守卫控制需要登录的页面访问

## 6. 路由设计原则

### 6.1 模块化设计

路由按照功能模块进行划分，便于维护和扩展：

- 核心功能模块
- 业务功能模块
- 用户中心模块
- 社交互动模块

### 6.2 语义化路径

路由路径设计遵循语义化原则，便于理解和维护：

- 使用名词命名资源
- 使用动词命名操作
- 参数化路径便于识别动态内容

### 6.3 一致性原则

路由设计保持一致性，提高开发效率和用户体验：

- 统一的命名规范
- 一致的参数传递方式
- 统一的页面跳转逻辑
